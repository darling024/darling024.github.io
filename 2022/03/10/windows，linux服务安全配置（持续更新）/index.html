windows，linux服务安全配置（持续更新）

2022-03-10- 2022-03-12

知识点：linux，windows服务安全配置

linux

Apache安全加固

后台进程：httpd
默认使用的端口：80（http），443（https）
配置文件的路径：一般实在根目录下面的etc/httpd或者etc/apache2
以特定的用户去运行服务，不要使用系统的管理员账号启动apache，防止收到越权使用造成的非法工具，修改的话：user apache group apache
记住一定要重启一下apache的服务不然不起效。

隐藏apache的banner信息，将ServerTokens OS修改成ServerTokens Prod
意思是网页在出现错误的页面的时候不显示服务器操作系统的名称
将ServerSignature On修改成ServeerSignature Off
意识是不回显apache的版本信息

禁止目录浏览，将Options indexes FollowSymLinks修改成Options FollowSymLinks

限制IP的访问，
<Directory “/var/www/html/aa/“>
Options Indexes MultiViews FollowSymLinks
AllowOverride AuthConfig
AuthType Basic
AuthName “testuser’s paasword”
AuthUserFile /usr/local/etc/passwd.httpd
Require user testuser
Order allow,deny
Allow from 172.16.1.0

限制禁止访问的文件夹，比如说是网站的后台目录
<Directory “/var/www/html/33”>
Deny from all

防止appache的解析漏洞
一般来说，apache对于文件名的解析是从后向前解析的，而且apache在解析一个文件的时候，会一直解析到一个它认识的文件后缀名，如果说是在一个网站的后台目录下面有一个1.php.test的文件，apache在解析的时候，当解析到test的时候会接着向前解析，一直到解析到php的时候，就会将这个文件判定为php类型的文件而不是原本的test类型文件
禁止httpd解析index.php.jpg的文件
在apache的服务配置文件中
<FilesMatch .php$>
SetHandler application/x-httpd-php

<FilesMatch “.phps$”>
SetHandler application/x-httpd-php-source

防止apache的服务解析
在配置文件中提娜佳一下的内容来组织apache服务去解析这种文件
<FilesMatch .php$>
SetHandler application/x-httpd-php

<FilesMatch “.phps$”>
SetHandler application/x-httpd-php-source

错误的页面重定向
在访问一个网页的时候难免会出现一些无法访问的子网页，或者说是这个网页里面的站，这个时候就会给我们跳出一个404或者是其他的错误网页的界面，一般是在htaccess文件中加入
ErrorDocument 400 /custom400.html
ErrorDocument 401 /custom401.html
ErrorDocument 403 /custom403.html
ErrorDocument 404 /custom404.html
ErrorDocument 405 /custom405.html
ErrorDocument 500 /custom500.html Customxxx.html 为要设置的错误页面。
添加之后记得重启不然是无效的。

日志文件的设置
LogLevel notice //日志的级别
ErrorLog logs/error_log //错误日志的保存为位置
CustomLog logs/access_log combined //访问日志
CustomLog指令设置访问日志的文件名和位置
LogFormat设置日志的格式，LogLevel用于调整记录在错误日志中的信息的详细程度，建议设置成notice

拒绝服务防范
根据自己配置的需要，然后设置session的时间去防止拒绝服务的攻击
Timeout 10 //客户端与服务器端建立连接前的时间间隔
KeepAlive On
KeepAliveTimeout 15 //限制每个session的保持时间是15秒

禁用CGI
如果运行的服务器不需要CHI陈旭，可以直接把CGI干掉
修改配置文件中的，将cgi-bin目录的配置和模块全部都注释掉
//LoadModule cgi_module modules/mod_cgi.so
ScriptAlias /cgi-bin/ “/var/www/cgi-bin/“
<Directory “/var/www/cgi-bin”>
AllowOverride None
Options None
Order allow,deny
Allow from all
// （全部注释掉）

防止SQL注入
这是一个比较常见的服务配置的漏洞了，所以说是一定要进行加固防御的，小则网站后台被入侵重则整个服务器都可以沦陷
将php.ini文件中的magic_quotes_gpc = Off改成magic_quotes_gpc = On

关闭远程文件打开
allow_url_fopen = off
关闭为了防止黑客远程包含的漏洞
